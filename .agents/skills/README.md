# .agents/skills/ — 项目级技能目录

本目录集中管理 Claw（项目 AI 助手）可调用的所有**技能（Skills）**。

---

## 目录结构约定

```
.agents/skills/
├── README.md               # 本文件：目录规范
└── <skill-name>/           # 每个技能独立目录，名称使用 kebab-case
    ├── README.md           # 技能说明（必须）：功能、触发方式、示例
    └── ...                 # 可选附属文件（脚本、配置、模板等）
```

---

## 技能文件规范

每个技能目录内的 `README.md` 必须包含以下内容：

```markdown
# 技能名称

## 功能描述
一句话说明这个技能做什么。

## 触发关键词
列出用户可以用哪些词语/句式来调用本技能。

## 使用说明
详细步骤或参数说明。

## 示例
给出 1~3 个典型用法示例。

## 依赖
说明本技能依赖的工具、权限或外部服务（无则填"无"）。
```

---

## 技能发现机制

Claw 在每次会话开始时，应执行以下步骤发现可用技能：

1. **扫描目录**：列出 `.agents/skills/` 下的所有子目录。
2. **读取描述**：读取每个子目录的 `README.md`，提取「功能描述」与「触发关键词」。
3. **注册到上下文**：将已发现的技能名称与简介保存在当前会话记忆中，作为可调用能力列表。

---

## 技能安装约定

- **添加新技能**：在 `.agents/skills/` 下新建独立目录，按上述规范编写 `README.md` 即视为"已安装"。
- **禁用技能**：在目录名前加 `_disabled_` 前缀（如 `_disabled_web-fetch`），Claw 将忽略该技能。
- **删除技能**：直接删除对应目录。

无需执行任何安装命令，约定即机制。

---

## 技能使用约定

调用技能时，Claw 应：

1. 识别用户意图是否匹配某个技能的触发关键词。
2. 读取该技能的 `README.md` 获取最新使用说明。
3. 按照说明执行，并在执行后更新会话日志。

若技能执行失败或说明不完整，应主动告知用户并请求补充。
